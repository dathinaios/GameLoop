Vehicle


superclasses: Entity, MobileEntity


Vehicle includes all the variables required for implementing steering behaviours. To learn more about 

steering behaviours refer to: 


Reynolds, C. W. (1999) Steering Behaviors For Autonomous Characters, in the proceedings of Game Developers Conference 1999 held in San Jose, California. Miller Freeman Game Group, San Francisco, California. Pages 763-782.

can be found at:


http://www.red3d.com/cwr/steer/


*new(world, position, radius, mass, velocity, side, heading, maxSpeed, maxForce, maxTurnRate)

world is an instance of EntityManager or EntityManager2. See the above paper for the rest of the variables. 

I havent actually used the variables side, heading and maxTurnRate that are part of the local coordinate

system but they may be essential for the inplementation of certain steering behaviours so I included them.


update

update the entity. Currently using Euler integration. If you need a better integrator make a subclass and

code the update method. The difference of the integration method from MobileEntity is the constrain in maxSpeed.
