EntityManager2



EntityManager2 functions like EntityManager but you can add to it 3 types of entities: free, static and mobile.

When adding a free Entity the manager doesn't include it when checking for collisions. Mobile entities are 

checked against each other and with static ones while static are not checked against each other. This saves 

processing time by reducing the number of checks while the spatial hashing index becomes more efficient 

by registering the static entities one time only, at the time of creation. 


*new(aSpatialHashing)

creates a new EntityManager passing it the spatial Index.


Instance Methods


addFree(a Entity)

add a free entity to the manager. Used internally by the Entity class

and its subclasses.


removeFree(a Entity)

remove a free entity to the manager. Used internally by the Entity class

and its subclasses.


addStat(a Entity)

add a static entity to the manager. Used internally by the Entity class

and its subclasses.


removeStat(a Entity)

remove a static  entity to the manager. Used internally by the Entity class

and its subclasses.


addMob(a Entity)

add a mobile entity to the manager. Used internally by the Entity class

and its subclasses.


removeMob(a Entity)

remove a mobile entity to the manager. Used internally by the Entity class

and its subclasses.


update

update all entities


clear

clear all entities


refreshIndex1

refreshIndex2

refresh can not happen simple by clearing buckets as in EntityManager because we need to keep

the registered static elements. It also needs to happen in two stages (one before and one after update)

because SpatialHashing needs to know the positions of the entities in order to unregister them (refreshIndex1).

Only then we can update the positions and register them again.


collisionCheck

check for collisions and notify entities


Examples


//The main loop takes a slightly different form:


(


//create he manager and pass it the spatial index instance.

~entityManager = EntityManager2(SpatialHashing(20, 20, 0.5));


(

/// START ///

{

inf.do{

//unregister the mobile entities

~entityManager.refreshIndex1;

//update all entities (move them)

~entityManager.update; 

//reregister the mobile entities after the motion

~entityManager.refreshIndex2;

//check for collisions

~entityManager.collisionCheck;

//**************************\\

//usually here you would render in visuals or sound

//**************************\\

~entityManager.dt.wait; //update at 20 fps

}

}.asRoutine.play

)
