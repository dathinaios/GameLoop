EntityRepresentation



EntityRepresentation is the base class for all entity representations. the representations are the 

place where the entities are rendered into sound, visuals or anything else you'd like. An entity can 

have multiple representations as in most of the examples where the entities are translated as both 

sound and visuals.



*new(entity)

create it and pass it the entity. note that the entity won't know anything

about its representation according to the MVC pattern. The representation 

will be querring the entity for its state.


add

add it to the RepresentationManager


remove

remove it from the RepresentationManager


update

calls the getData method


getData

Should be implemented in the subclass to get the data from the entity and

assign them to the relevant parameters. Called every update.


run

this method runs the rep but does not add the representation to the manager. 

Seperating the activation from the running can be usefull when you use bundles.

Typically something like that:


Routine{

rep.run; //run the representation bundle (is going to play after latency)

latency.wait; 

entity.activate;//activate the entity after latency to account for the  bundle

  //the activate method is going to call the activate to the rep


}.play(clock);


activate

this method adds the rep to the manager but does not run the representation. 

It is called by the entity method activate to all its representations.


activateRun

activate and run at the same time.
