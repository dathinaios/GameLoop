EntityManager



EntityManager is a 'world' for Entities. When an Entity is created it is added to an EntityManager. the responsibilities of

the manager are: (1) updating the entities (the time between updates is stored in the managers class variable dt). 

(2) checking for collisions using an instance of SpatialHashing and sending the .collision msg to the entities. 


See also: EntityManager2


*new(aSpatialHashing)

creates a new EntityManager passing it the spatial Index.


Instance Methods


add(a Entity)

add an entity to the manager. Used internally by the Entity class

and its subclasses.


remove(a Entity)

remove an entity to the manager. Used internally by the Entity class

and its subclasses.


update

update all entities


clear

clear all entities


refreshIndex

clear and recalculate the spatial index. This step is needed in the main loop

to register the new positions of the entities in the spatial hashing grid (see SpatialHashing)


collisionCheck

check for collisions and notify entities


Examples


//The main loop:


(


//create he manager and pass it the spatial index instance.

~entityManager = EntityManager(SpatialHashing(20, 20, 0.5));


(

/// START ///

{

inf.do{

//update all entities (move them)

~entityManager.update; 

//refresh the spatial index after the motion

~entityManager.refreshIndex;

//check for collisions

~entityManager.collisionCheck;

//**************************\\

//usually here you would render in visuals or sound

//**************************\\

~entityManager.dt.wait; //update at 20 fps

}

}.asRoutine.play

)
